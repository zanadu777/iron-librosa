Phase-Vocoder: Promote Rust dispatch to default

This commit enables the Rust phase-vocoder kernel by default while
maintaining full backward compatibility via the prefer_rust parameter.

MOTIVATION
==========

The Rust phase-vocoder kernel provides 1.5–2.0× performance improvement
over the Python reference implementation while maintaining numeric parity
(within 1e-5 for float32, 1e-11 for float64).

TECHNICAL CHANGES
=================

1. librosa/core/spectrum.py
   - Add prefer_rust=True parameter to phase_vocoder()
   - Remove IRON_LIBROSA_ENABLE_RUST_PHASE_VOCODER env var gate
   - Simplified dispatch: if prefer_rust and RUST_AVAILABLE and ...

2. src/phase_vocoder.rs
   - Use .round_ties_even() for phase wrapping (NumPy-compatible)
   - Both f32 and f64 paths aligned for parity

3. tests/test_features.py
   - Remove @pytest.mark.xfail from parity test (now expected to pass)
   - Update dispatch tests: default uses Rust, fallback works
   - Add test_phase_vocoder_dispatch_prefers_rust_by_default
   - Add test_phase_vocoder_dispatch_fallback_with_prefer_rust_false

4. Documentation
   - PHASE_VOCODER_FIX.md: root cause & fix analysis
   - PHASE_VOCODER_PARITY_CHECKLIST.md: promotion criteria
   - PHASE_VOCODER_PROMOTION_COMPLETE.md: promotion summary
   - RELEASE_NOTES_PHASE_VOCODER.md: user-facing notes
