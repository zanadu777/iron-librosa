### Key Innovation

The dispatch heuristic intelligently avoids the performance pitfall that would occur from naive batch parallelism:

**Problem:** Nested rayon at both batch and element level causes task scheduling overhead

**Solution:** Only parallelize when:
1. Batch count is large enough (>= 4) to justify thread pool overhead
2. Per-batch work is small (< 150K) so inner kernels stay sequential
3. This avoids nested contention while maintaining cache locality

## How to Use

The batch functions now automatically use:
1. **Parallel dispatch** for: batch >= 4 AND per_batch_elements < 150,000
2. **Sequential dispatch** otherwise

Example:
```python
import numpy as np
import librosa

# Small-work batch (will use parallelism)
S = np.random.rand(8, 256, 100).astype(np.float32)  # Triggers parallelism
H, P = librosa.decompose.hpss(S, kernel_size=(31, 31))

# Large-work batch (sequential, inner may parallelize)
S = np.random.rand(8, 1025, 600).astype(np.float32)  # Sequential dispatch
H, P = librosa.decompose.hpss(S, kernel_size=(31, 31))
```
