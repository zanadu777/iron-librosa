# IMPLEMENTATION DETAILS

## Dispatch Strategy

The adaptive dispatch mechanism uses a two-condition check:

```
if (batch_size >= 4) AND (per_batch_elements < 150,000):
    use_batch_level_parallelism()
else:
    sequential_batch_processing()
```

### Why These Thresholds?

**Batch Size >= 4:**
- Rayon thread pool has initialization overhead
- Need sufficient independent work to amortize this cost
- 4 batches = minimum viable for typical 8-core systems
- Tuned empirically to maximize speedup

**Per-Batch Elements < 150,000:**
- PAR_THRESHOLD (inner parallelism) = 200,000 elements
- By keeping per-batch < 150K, inner row-level kernels stay sequential
- Avoids nested rayon task scheduling overhead
- Allows batch rayon to distribute independent sequential jobs
- Maximizes cache locality within each batch
